# 2026–2027 主线龙头投资体系 V2.0 · 作战手册

> 生效日期：2026-07-17
> 定位：体系的**总量与选择层**，位于战略层（组合总纲）与执行层（交易操作系统）之间。回答三个问题：现在是什么市场阶段？该有多少总仓位？只允许从哪些股票里选？
> 内含并正式收编「GPT 趋势投资体系 V1.0」四问铁律（第三章）。
> 冲突裁决：本手册与其他文件冲突时——卖出与止损一律以执行层《交易操作系统》第二章为准（优先级最高）；名单与评分以战略层《组合总纲》为准；总仓位与买入许可以本手册为准。

## 第〇章 第一原则

> **不会因为恐慌而卖，也不会因为跌了就抄底。**

这两句话是整个体系的第一原则。它的规则化表述：

- 卖出只能由执行层规则触发（均线、熔断、硬止损），恐慌不是触发条件。
- 买入只能由「止跌确认 + 预设价格区间 + 四问全过」触发，跌幅本身不是触发条件。**跌26%不是买点，跌26%以后止跌才可能是买点——这是两件完全不同的事。**

## 第一章 决策顺序（永久固定）

```
第一步：判断市场阶段（上升 / 震荡 / 下跌）
第二步：决定总仓位（由阶段决定，不由个股决定）
第三步：决定标的（只从主线龙头梯队里选）
```

禁止倒序操作：不允许因为「某只股票很便宜」而倒推出「所以该加仓」。即使市场继续跌，我们也不推翻昨天的判断——只用每天的数据验证规则，不用盘面创造新规则。

## 第二章 市场阶段判定与仓位管理模型

### 2.1 阶段判定（以约定基准指数为准：科创50 + 半导体行业指数，双指数共振才改判）

| 阶段 | 判定标准（收盘价口径） | 目标总仓位上限 |
|---|---|---|
| 上升期 | 指数站上20日线且20日线向上，60日线向上 | **80%** |
| 震荡期 | 指数在20日线上下反复，60日线走平 | **60%** |
| 下跌期 | 指数收于20日线下方且20日线向下 | **40%** |

- 阶段改判需要**连续2个交易日**收盘满足新阶段标准，单日不改判。
- 阶段判定每天写入盘前四问，全年留痕。

### 2.2 仓位模型的执行规则（关键条款）

1. **仓位上限只约束买入，不强制卖出。** 若因阶段下调导致当前仓位超过上限（如现在：下跌期上限40%，实际仓位64%），处理方式是：**冻结全部加仓**，存量仓位仍按执行层卖出规则自然退出，**禁止**为了达标而恐慌性一次砍仓。这就是「不追杀」的制度化。
2. 阶段上限与执行层第四章硬约束（单股12%、板块30%、主题45%、现金≥10%、单日新增≤5%）同时生效，取最严者。
3. 上升期确认后，仓位从当前水平向80%靠拢也必须走执行层第三章买入规则，分批进行。

## 第三章 四问铁律（GPT 趋势投资体系 V1.0，正式收编）

任何一笔买入/加仓，四问必须全部通过，任何人不得破例，**包括提出建议的我**：

| 问 | 内容 | 不通过的处理 |
|---|---|---|
| ① | **是主线吗？** | 不是主线 → 不买 |
| ② | **是龙头吗？（赛道前三）** | 不是前三 → 不买 |
| ③ | **下跌趋势结束了吗？** | 没有结束 → 不接飞刀 |
| ④ | **资金回来了吗？** | 没有资金 → 不加仓 |

- ①②由战略层回答（主线识别模型 + 评分排名），③④由数据回答（第六章止跌确认 + 第九章资金指标）。
- 四问记录在每笔买入的盘前四问日报中，缺一问的买入视为违规操作，计入周度偏离次数。

## 第四章 主线识别模型（哪些行业值得长期重仓）

一个行业被认定为「主线」，必须同时满足四个条件，且每季度复核：

1. **政策**：有国家级、持续性的政策支持（不是一次性口号）；
2. **产业趋势**：渗透率/国产化率处于快速提升期，空间可量化；
3. **业绩兑现**：板块内龙头公司连续 ≥2 个季度订单或利润验证逻辑；
4. **资金持续性**：机构持仓趋势性增加，不是单周游资行为。

**当前认定的主线（V2.0 初始状态）：**

- ✅ AI算力（国产GPU、服务器、光互联）
- ✅ 半导体自主可控（设备、材料、制造）
- ⏸️ 机器人：**暂不认定**。真正龙头出来以后再决定，目前不满意。列为「候补主线」，只研究不建仓。

主线的增删按战略层修订流程（季度评审）执行，禁止盘中临时认定新主线。

## 第五章 龙头筛选标准（为什么选它，而不是第二名）

在主线内，只有同时满足以下条件才有「龙头」资格：

1. 细分赛道市占率/技术代差位列**前三**（四问之②的量化口径）；
2. 战略层评分 ≥ 80 分，且位列赛道前二；
3. 盈利质量：毛利率不低于赛道中位数，经营现金流为正或订单充分覆盖;
4. 机构股东名单中存在持续加仓的长线资金。

第二名值得买的唯一情形：与第一名存在明确的差异化定位（不同客户/不同工艺），且评分差 <5 分。否则资金只给第一名。

## 第六章 趋势确认规则（什么时候允许买）

机构的节奏是：下跌 → 下跌 → 缩量 → 止跌 → 再买。**宁可少赚10%，不接飞刀。** 因此规定止跌确认三要素（个股口径，缺一不可）：

1. **缩量**：连续 ≥2 日成交量低于5日均量的 70%；
2. **止跌**：收盘价 ≥3 个交易日不再创调整新低；
3. **确认**：放量收复5日线（首次试探仓位），或放量收复20日线（正式右侧仓位）。

配套规则：

- 三要素确认前，预设价格区间即使到价也**只挂观察、不下单**——到价 + 止跌确认，才构成买入触发。
- 首次试探仓位 ≤ 该股计划仓位的 1/3；收复20日线后才允许第二笔。
- 「抢第一根反弹」列为违规行为清单第一条。

## 第七章 加仓规则（什么时候加，什么时候绝不加）

**允许加仓，须全部满足：**

1. 市场阶段为上升期或震荡期（下跌期全面冻结加仓，见第2.2条）;
2. 该股浮盈状态（成本线上方）——只在盈利的仓位上加码;
3. 四问全过，且不突破单股12%上限;
4. 距上一笔买入 ≥3 个交易日（禁止连日追买）。

**绝不加仓清单：**

- ✗ 下跌途中摊低成本（「补仓摊平」永久禁止）;
- ✗ 处于执行层卖出规则执行过程中的股票;
- ✗ 通信板块（整改期内）;
- ✗ 当日已有一笔买入的同一股票。

## 第八章 减仓与止损

全部沿用执行层《交易操作系统》第二章默认模板与个股规则卡，本手册不另立卖出规则、也无权放宽。唯一补充：

- 市场阶段被改判为「下跌期」当日起，所有持仓的20日线减仓条款计数**从宽改严**：「连续2日」改为「1日」触发（下跌期对破位零容忍；阶段改回震荡/上升后自动恢复2日）。

## 第九章 每天只检查的5个指标

每日盘前四问日报新增固定栏目，只看这5项，看完即止：

| # | 指标 | 回答的问题 |
|---|---|---|
| 1 | 基准指数 vs 20日/60日线 | 市场阶段是否需要改判？ |
| 2 | 持仓股是否触发执行层卖出条款 | 今天必须卖什么？ |
| 3 | 目标股是否进入预设买入区间且完成止跌三要素 | 今天允许买什么？ |
| 4 | 板块成交量与主力/北向资金方向 | 资金回来了吗？（四问之④） |
| 5 | 仓位 vs 阶段上限、板块/单股上限 | 有没有违规敞口需要冻结或整改？ |

这5项之外的信息（新闻、传闻、盘中波动）一律进数据库转化为字段变更，不进当日决策。

## 第十章 主线龙头梯队（观察池优先级）

> 梯队 = 研究与建仓的优先级排序，不是买入许可。买入许可只来自四问 + 止跌确认 + 预设区间。名单细节与晋降级见《AI产业链全景图与观察名单》。

**第一梯队（战略核心，优先建档、优先评分）：**
海光信息、中际旭创、新易盛、北方华创、中芯国际、寒武纪（条件：趋势重新转强后才具备买入资格）

**第二梯队：**
澜起科技、兆易创新、中微公司、拓荆科技、汇川技术

**第三梯队（机器人）：**
真正龙头出现后再决定，目前空缺。只建档研究，禁止建仓。

梯队与战略层评分的关系：梯队是先验排序，季度评分完成后**以评分为准**重新排定；评分与梯队冲突时，评分胜。

## 附录：违规行为清单（周度考核用）

1. 抢第一根反弹（未完成止跌三要素即买入）
2. 下跌途中摊低成本
3. 四问未全过即买入
4. 下跌期加仓
5. 恐慌性卖出（无执行层条款触发的卖出）
6. 盘中临时认定新主线或临时修改规则
7. 因单日盘面推翻前日阶段判定（未满足连续2日改判标准）

每周复盘统计违规次数，目标为 0。盈亏不作为周度考核指标。
