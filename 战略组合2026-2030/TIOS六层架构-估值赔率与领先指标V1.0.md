## 第一章 新决策流程（永久生效）

```
旧流程（废止）：看到新闻 → 分析新闻 → 决定买卖
新流程（生效）：领先指标变化 → 更新产业判断 → 更新估值与赔率 → 最后才决定交易
```

**交易永远放在最后一步。** 任何跳过前三步直接到交易的提议，自动无效。

## 第二章 估值引擎（Valuation Engine）——补第一缺口

**「便宜」「贵」「位置」从此必须是数字，不是感觉。**

### 2.1 必备字段（每只核心公司，缺一即无价格资格）

| 字段 | 说明 |
|---|---|
| PE（TTM与2026E/2027E前瞻） | 前瞻PE用一致预期盈利 |
| PEG | 前瞻PE ÷ 未来两年盈利CAGR |
| EV/EBITDA | 重资产公司（制造/设备）主锚 |
| PS | 高研发期/利润未释放公司辅锚 |
| 历史百分位 | 自身3年与5年PE（或PB）分位 |
| 未来盈利 | 2026E/2027E一致预期净利与更新日期 |
| 五年ROE | 质量锚，验证「冠军」成色 |

### 2.2 与星级区间的对接

CTC第四章星级区间的判定输入即为本引擎输出：★★★★★ = PEG<0.8且PE低于3年中位数20%以上；★ = PEG>1.6或PE处90分位以上。**任何买点必须来自估值+趋势，不是K线。**

### 2.3 示范测算（海光信息，粗算口径，标注为示例、待精确回填）

- 已知：Q1净利6.87亿、EPS 0.22元 → 总股本约31.2亿股；现价300.64 → 市值约9,380亿
- 中报预告全年隐含：2026E净利粗算38~45亿（H1中值17.6亿×季节性系数，待一致预期校准）
- **前瞻PE粗算：约210~245倍；盈利CAGR约45% → PEG约4.7~5.4**
- 结论示范：按引擎口径，海光当前价格处于**★（高估值区）**——「300附近舒服」的说法在数字面前不成立，300只是比310「跌过一点」，不是「便宜」。**这正是估值引擎存在的意义：它会说出我们不想听的话。**
- 待办：用券商一致预期替换粗算；北方华创（680）、中微、中芯同步建模（本周三前）

## 第三章 赔率引擎（Risk Reward Engine）——补第二缺口

**以后不是「喜欢」，而是 Risk : Reward。**

### 3.1 计算口径

- **上行空间** = （目标价 − 现价）/ 现价。目标价 = 2027E一致预期盈利 × 历史中位前瞻PE（保守）与75分位PE（乐观）两档
- **下行空间** = （现价 − 防守价）/ 现价。防守价 = max（硬止损位，★★★★★区间下沿，关键技术支撑）
- **赔率 RR = 上行 ÷ 下行**

### 3.2 阈值（硬性）

| RR | 动作 |
|---|---|
| ≥ 3:1 | 允许第一笔建仓（仍需两证+三道闸） |
| 2~3:1 | 只允许对已有盈利仓位加码 |
| 1.5~2:1 | 持有不加 |
| < 1.5:1 | **禁止买入**；持仓进入减仓评估 |

### 3.3 强制记录

每笔买入指令必须附RR计算过程（目标价依据+防守价依据），存档于日志。示例格式：「北方华创：上行30% / 下行10% → RR 3:1 → 可买」；「新易盛：上行20% / 下行25% → RR 0.8 → 禁止」。

## 第四章 领先指标看板（Leading Indicator Dashboard）——补第三缺口

**财报、Capex、ETF都是滞后信息。真正的领先指标提前两个月。每个产业只保留三个，天天看，不用等财报，不用看几十条新闻。**

### 4.1 首版看板

| 主线 | 指标① | 指标② | 指标③ |
|---|---|---|---|
| **AI基础设施** | 台积电CoWoS产能与扩产节奏 | HBM价格/订单/SK海力士库存 | 四大云厂订单与Capex先行信号（英伟达Lead Time为代理） |

| **光模块/高速互联** | 800G/1.6T排产与月度出货（产业链调研口径） | 北美云厂网络架构招标（CPO渗透信号） | 上游光芯片（EML/硅光）交期 |
| **存储链** | DRAM/NAND现货价周变动 | 三星/海力士稼动率与扩产公告 | 模组厂库存周数 |

### 4.2 运行规则

1. 每个指标登记：当前读数、方向（↑/→/↓）、更新日期、数据源；**每周五收盘后统一更新**，重大变化当日标记。
2. 指标方向变化 → 触发产业判断复核 → 触发估值/赔率重算 → 最后才可能产生交易（第一章流程）。
3. 指标增删每季度评审，任何时候每主线保持恰好三个——多于三个是噪音，少于三个是盲区。

## 第五章 六层架构定稿（TIOS 完整体）

```
① 战略层   CTC（五年）             —— 谁配得上钱
② 战术层   作战地图（3个月）        —— 现在打哪个方向
③ 领先指标层 Dashboard（每周）      —— 产业实际发生了什么（新增）
④ 估值层   Valuation Engine        —— 到底贵不贵，数字说话（新增）
⑤ 赔率层   Risk Reward Engine      —— 值不值得冒险，RR说话（新增）
⑥ 执行层   Trend OS（每笔）        —— 何时进出，纪律说话
```

信息自上而下流动，交易许可自下而上收敛：六层全部通过，一笔交易才能发生。

## 第六章 规则卖出的失效条件条款（对执行层的唯一修订，窗口前以附则生效）

原表述「周一必须卖」修订为：**「周一执行卖出计划，除非出现预先定义的失效条件」**。

**失效条件（穷举，不得临时增补）：**

1. 该股开盘前发布公司层面重大公告（业绩预告上修、重大合同、回购方案落地、调查撤销/和解等，须有公告编号）；
2. 海外发生改变中期预期的重大事件（如云厂商提前披露超预期Capex、行业性政策转向，须有可核实信源）；
3. 交易所层面熔断/停牌等导致无法正常执行。

**防滥用护栏（与条款同权重）：**

- 失效条件必须**开盘前书面援引**，注明公告编号/信源，「盘面感觉」不构成失效；
- 效果是**推迟至当日收盘复核**，不是取消——收盘后条款仍满足则次日照旧执行；
- 每次援引计入周度复盘；事后证明不成立的援引，按违规记录。
