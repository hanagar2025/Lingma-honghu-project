# 🚀 快速启动指南

## ⚠️ 问题说明

由于端口 5000 被占用，后端服务可能无法正常启动。

## 🔧 解决方案

### 方法一：使用不同端口启动后端

在终端运行：

```bash
cd backend
PORT=3001 npm run dev
```

然后访问：
- 前端: http://localhost:5173
- 后端: http://localhost:3001

### 方法二：修改配置文件

编辑 `backend/.env` 文件：

```env
PORT=3001
```

然后启动：
```bash
cd backend
npm run dev
```

### 方法三：关闭占用端口 5000 的服务

查找并关闭占用端口的进程：

```bash
# 查找占用 5000 端口的进程
lsof -i :5000

# 杀死进程（替换 PID 为实际进程ID）
kill -9 <PID>
```

---

## ✅ 验证服务是否运行

### 1. 检查前端服务

打开浏览器访问：**http://localhost:5173**

应该能看到登录页面。

### 2. 检查后端服务

在终端运行：

```bash
curl http://localhost:3001/health
```

应该返回：
```json
{"status":"ok","timestamp":"..."}
```

## 📞 遇到问题？

### 前端页面空白

1. 检查浏览器控制台（F12）
2. 查看是否有错误信息
3. 确认后端服务正常运行

### 后端连接失败

1. 检查后端是否在运行
2. 确认端口号正确
3. 查看 `backend/.env` 配置

### 数据库连接失败

1. 确认 MySQL 服务运行：
   ```bash
   brew services list | grep mysql
   ```

2. 测试连接：
   ```bash
   mysql -u root
   ```
