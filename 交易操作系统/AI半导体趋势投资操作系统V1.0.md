# AI半导体趋势投资操作系统 V1.0

> 生效日期：2026-07-17
> 状态：正式生效（替代此前所有口头约定和盘中临时决策）
> 修订权限：见第八章「规则修订流程」。盘中、当日、根据当天行情，一律不得修改本文件。

---

## 序言：本系统存在的理由

过去一周暴露的问题不是分析能力，而是决策一致性：

- 「不急，观察一天」→「再观察半天」→「等收盘确认」——这是确认偏误，不是趋势交易。
- 「如果昨天上午知道会这样，我应该卖」——这是事后诸葛亮。真正的交易系统不能根据结果修改昨天的规则。
- 通信仓位半个月前就超过风控上限，却没有被强制纠正。
- 讨论了无数次「什么时候买」，几乎没有成文的「什么时候必须卖」。

本系统的目标只有一个：**让每天的操作在开盘前就已经被规则决定，而不是在盘中被情绪决定。**

从今天起，我们评价的不是谁猜对了行情，而是谁坚持执行了系统。

---

## 第一章 身份定义：我们是什么投资者

**明确回答：我们不是趋势投资者。当前组合（中际旭创、新易盛、海光、北方华创、中微、兆易、澜起……）的实际风格是「高Beta成长长期持有」——没有止损、没有右侧加仓、没有趋势退出。**

V1.0 的身份定义为：

> **选股用成长逻辑，仓位管理用趋势规则。**

- **选股层（买什么）**：只买 AI 算力、半导体设备/材料/设计、光模块等有产业逻辑和业绩支撑的成长股。入池标准见第四章。
- **仓位层（拿多少、何时进退）**：完全服从均线趋势规则和风控上限，不服从「感觉」「大概率」「再观察一下」。

这意味着两条铁律：

1. 再看好的公司，触发退出规则也必须退出（可以退出后按买入规则重新买回）。
2. 再恐慌的下跌，未触发规则就不卖（「今天什么也不做」是合法且常见的答案）。

## 第二章 卖出规则（必须在持有期间提前成文）

### 2.1 默认卖出模板（适用于所有持仓，除非该股有专属规则）

| 触发条件 | 动作 | 备注 |
|---|---|---|
| 收盘价跌破20日均线，且连续2个交易日收于其下 | 减持该股当前仓位的 20% | 第2个交易日收盘后、次日开盘执行 |
| 收盘价跌破60日均线 | 减持至原仓位的 40% 以下（即累计至少减60%） | 触发当日收盘确认，次日开盘执行 |
| 收盘价跌破120日均线 | 全部退出 | 无商量、无例外、不等反弹 |
| 单日跌幅 ≥ 12%（含跌停） | 次日开盘先减 30%，剩余部分按均线规则执行 | 熔断式减仓，防止澜起式深跌被动扛单 |
| 个股自持仓成本回撤 ≥ 25% | 全部退出 | 硬止损，优先级高于一切「基本面没变」的理由 |

**执行说明：**

- 所有触发以**收盘价**为准，盘中破位不算触发（防止盘中反复打脸），但触发后**次日开盘必须执行**，不允许「等等看会不会收回来」。
- 「连续2日」的计数一旦开始，中间收回均线则清零重计。
- 减仓后若股价重新站回对应均线并满足买入规则，可按第三章重新加回，这不是打脸，这是系统的正常成本。

### 2.2 个股专属规则示例（持仓每只股票都必须有一份，写在个股规则卡中）

**海光信息：**

- 收盘跌破20日线连续2日 → 减 20%
- 收盘跌破60日线 → 减 40%
- 收盘跌破120日线 → 全部退出

**规则卡要求：** 每只持仓股票必须在建仓当日或本系统生效后 3 个交易日内完成规则卡（见附录模板），未完成规则卡的股票**禁止加仓**。

### 2.3 卖出规则的唯一豁免

无。基本面暴雷、财报造假、重大利空等情形只会**提前**卖出（直接全退），不会成为**推迟**卖出的理由。

## 第三章 买入规则（必须在下跌/启动之前提前成文）

### 3.1 原则

- 禁止盘中临时决定买入。所有买入必须命中**事先写好的**价格/形态触发条件。
- 每只候选股票最多设 3 个分批买入触发点，全部提前写入规则卡。

### 3.2 左侧分批模板（示例：北方华创）

| 批次 | 触发条件 | 买入金额 |
|---|---|---|
| 第一批 | 自阶段高点回撤 20% | 计划仓位的 1/3 |
| 第二批 | 自阶段高点回撤 30% | 计划仓位的 1/3 |
| 第三批 | 放量反包阳线（成交量 ≥ 5日均量1.5倍，收盘收复前一根阴线实体） | 计划仓位的 1/3 |

### 3.3 右侧趋势模板

- 收盘站上20日线且20日线拐头向上 → 可建第一批；
- 放量突破前期平台/阻力位 → 可加第二批；
- 回踩20日线不破并企稳 → 可加第三批。

### 3.4 买入禁令

以下情形**禁止买入**，无论多么「便宜」：

1. 该股正处于第二章任一卖出规则的执行过程中；
2. 买入后将导致第四章任一仓位上限被突破；
3. 该股没有完成规则卡；
4. 触发理由是「今天跌了很多，感觉能反弹」。

## 第四章 仓位与风控上限（硬约束，优先级最高）

### 4.1 上限

| 层级 | 上限 | 说明 |
|---|---|---|
| 单一股票 | ≤ 12% | 按当日市值计 |
| 单一板块（如通信/光模块） | ≤ 30% | 按申万一级/明确约定的板块口径 |
| 单一主题（如 AI 半导体全链条） | ≤ 45% | 主题口径在规则卡中固定 |
| 单日新增买入 | ≤ 总资产 5% | 防止一天内情绪化梭哈 |
| 现金下限 | ≥ 10% | 任何时候保留弹药与容错 |

### 4.2 当前组合的存量违规与整改计划

**事实确认：通信板块当前约 65%，超出 30% 上限已超过半个月。这不是今天才违规，是早就违规，且提醒不够坚定。**

整改规则（一次性过渡条款，执行完毕后自动失效）：

- 自本系统生效日起 **10 个交易日内**，将通信板块降至 ≤ 45%；**20 个交易日内**降至 ≤ 30%。
- 减仓顺序：优先减已触发或最接近触发第二章卖出规则的个股；其次减单股占比超 12% 的部分。
- 反弹不是暂停整改的理由；整改期内通信板块**禁止任何加仓**。

### 4.3 组合级熔断

- 组合自最高净值回撤 ≥ 15%：总仓位强制降至 ≤ 50%，暂停所有左侧买入，仅保留右侧规则买入。
- 组合回撤 ≥ 25%：总仓位强制降至 ≤ 30%，进入「只卖不买」模式，直至组合收复 15% 回撤线。

## 第五章 退出机制（本系统最重要的一章）

过去最大的漏洞不是澜起、不是海光，而是**没有退出机制**——一直在讨论什么时候买，很少讨论什么时候必须卖。

因此规定：

1. **没有退出规则的持仓是违规持仓。** 规则卡中「退出」一栏为空的股票，视同触发整改，3 个交易日内补齐，否则按默认模板（第2.1节）强制适用。
2. **退出规则只能在上涨过程中修改**（且只能上移、收紧，例如把止损从120日线收紧到60日线），下跌过程中禁止放宽任何退出条件。
3. **「如果昨天知道会这样」永久禁语。** 复盘只允许问一个问题：当时规则触发了吗？触发了是否执行了？

## 第六章 每日运作流程

### 6.1 盘前（开盘前完成，这是每天唯一的决策输出）

每天早晨只回答四件事：

1. **今天有哪些股票触发了买入规则？**（列出股票、触发的具体条款、执行数量）
2. **哪些股票触发了减仓/卖出规则？**（同上）
3. **哪些股票仍然保持持有？**（未触发任何规则）
4. **哪些股票禁止操作？**（处于买入禁令或整改期）

如果没有任何触发，答案就是：**「今天什么也不做。」** 这是系统正常运转的标志，不是失职。

### 6.2 盘中

- 不做任何决策。盘中只记录，不操作（已在盘前确定的委托除外）。
- 盘中出现极端情况（个股 -12% 之类），对应动作也已写在第2.1节，仍然是「按既有规则执行」，不是临时决策。

### 6.3 盘后

- 更新各股与均线的位置、连续计数；
- 记录当日是否有规则触发、是否执行、有无偏离（偏离必须书面记录原因）；
- **不做任何行情预测。** 「今天怎么看」「明天可能涨/跌」这类输出永久取消。

## 第七章 建议输出标准（对 AI 助理的约束）

今后任何买、卖、加、减的建议，必须同时回答四个问题，缺一即无效：

1. **依据是什么？**（引用本系统的具体条款编号 + 触发数据）
2. **哪些条件会证明这个判断错了？**（明确的证伪条件）
3. **如果错了，什么时候退出？**（具体的退出触发点）
4. **这个规则是否适用于所有同类情况？**（是则通用，否则说明为何是特例并写入规则卡）

回答不了这四个问题时，标准答案是：**「没有足够依据做出交易决策。」** 这比一个看似合理、实则无规则支撑的建议更称职。

「感觉」「大概率」「再观察一下」不得作为主要依据出现在任何建议中。

## 第八章 规则修订流程

1. 本文件的任何修改，只能在**每月第一个周末**的例行复盘中进行，且修改自**下一个交易日**起生效，不追溯既往。
2. 触发重大事件（组合回撤超 15%、系统性风险等）可启动临时修订，但临时修订**当日不得执行任何因新规则产生的交易**，至少隔一个交易日生效。
3. 每次修订必须记录：改了什么、为什么改、用过去 3 个月的行情检验新旧规则的差异。
4. **禁止根据前一天的亏损结果反向修改规则。** 修改的理由必须是规则本身的逻辑缺陷，而不是「上次按规则做亏了」。

---

## 第九章 考核标准

每周复盘只统计两个数字：

- **执行率** = 已执行的触发次数 / 应触发次数。目标 100%。
- **偏离次数** = 未按规则执行（多做、少做、晚做）的次数。目标 0。

盈亏不作为周度考核指标。系统的盈亏在季度和年度层面检验（见第八章修订流程）。谁猜对了不重要，谁坚持执行了系统才重要。
