# 🤔 为什么需要这些工具？

## 📋 我们的股票决策小程序需要什么？

让我们用一个简单的比喻来解释：

想象你要开一家餐厅：
- **Node.js** = 你已经有的厨师（✅ 已安装）
- **Homebrew** = 采购工具（帮你买食材）
- **MySQL** = 仓库（存储菜品、订单、客户信息）
- **Redis** = 快餐窗口（快速取用常用数据）

---

## 🛠️ 工具1：Homebrew

### 是什么？
**Homebrew** 是 macOS 的包管理器，就像手机的"应用商店"。

### 为什么需要？
**没有 Homebrew**：你需要手动下载、解压、配置 MySQL 和 Redis，很麻烦。
**有了 Homebrew**：一行命令就能安装软件，自动处理依赖。

### 类比
- 像 **App Store** - 点一下就能安装应用
- 像 **pip**（Python）或 **npm**（Node.js）

### 安装方式
```bash
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
```

### 一次安装，终身使用
安装一次 Homebrew，以后所有软件都能用它轻松安装。

---

## 🗄️ 工具2：MySQL

### 是什么？
**MySQL** 是一个数据库管理系统，用来存储和管理数据。

### 为什么需要？
你的股票小程序需要存储大量数据：

| 数据 | 存储位置 | 为什么用 MySQL |
|------|---------|---------------|
| 用户账号信息 | MySQL | 安全、稳定、能加密密码 |
| 持仓股票 | MySQL | 持久化存储，关机也不丢 |
| 历史数据 | MySQL | 查询速度快 |
| 报表记录 | MySQL | 方便回溯和分析 |

### 类比
- 像 **Excel 表格** - 但功能强大千万倍
- 像 **银行数据库** - 安全可靠

### 没有 MySQL 会怎样？
❌ 数据只能存在内存里，重启就丢失
❌ 多用户访问会有问题
❌ 无法查询历史数据

### 有了 MySQL 会怎样？
✅ 数据永久保存
✅ 支持多用户同时使用
✅ 快速查询和分析
✅ 数据安全可靠

---

## ⚡ 工具3：Redis

### 是什么？
**Redis** 是一个内存数据库，也叫缓存数据库。

### 为什么需要？
股票数据变化很快，需要快速读写：

| 场景 | 为什么用 Redis |
|------|---------------|
| 股票实时价格 | 极速获取（毫秒级） |
| 用户登录状态 | 快速验证 |
| 热门股票排行 | 避免重复计算 |
| 数据缓存 | 减轻 MySQL 压力 |

### 类比
- 像 **电脑内存条** - 超级快
- 像 **ATM 机** - 快速取钱

### MySQL vs Redis

| 特性 | MySQL | Redis |
|------|-------|-------|
| 存储位置 | 硬盘 | 内存 |
| 速度 | 较快 | 极快 |
| 持久化 | ✅ 永久保存 | ⚠️ 可能丢失 |
| 用途 | 重要数据 | 临时/常用数据 |

### 使用场景对比

**股票价格**（用 Redis）：
```
用户 → Redis → 秒回（10ms）
```

**用户信息**（用 MySQL）：
```
用户 → MySQL → 较快回（50ms）
```
