# 页面加载问题解决指南

## 当前状态

✅ **前端服务**: 已启动，运行在 http://localhost:5174/

⚠️ **后端服务**: 需要手动启动

## 问题原因

后端服务（端口3000）没有运行，导致前端无法加载数据。

## 解决步骤

### 1. 启动后端服务

在项目根目录运行：

```bash
# 方式1: 使用启动脚本（推荐）
./start.sh

# 方式2: 手动启动
cd backend
npm run dev
```

### 2. 检查后端服务状态

启动成功后，你应该看到类似的输出：

```
服务器运行在端口 3000
环境: development
数据库连接成功
Redis连接成功
```

### 3. 验证服务

在浏览器中访问以下地址测试：

- **前端**: http://localhost:5174/
- **后端健康检查**: http://localhost:3000/health
- **后端API**: http://localhost:3000/api/auth/login

## 常见问题

### 问题1: 数据库连接失败

**错误信息**: `数据库连接失败`

**解决方案**:
1. 确保MySQL已安装并运行
2. 检查`.env`文件中的数据库配置
3. 创建数据库: `CREATE DATABASE stock_decision;`

### 问题2: Redis连接失败

**错误信息**: `Redis连接失败`

**解决方案**:
1. 确保Redis已安装并运行
2. 启动Redis: `redis-server`
3. 或者暂时禁用Redis功能（修改代码）

### 问题3: 端口被占用

**错误信息**: `Port 3000 is already in use`

**解决方案**:
```bash
# 查找占用端口的进程
lsof -i :3000

# 终止进程
kill -9 <PID>
```

### 问题4: 依赖未安装

**错误信息**: `Cannot find module`

**解决方案**:
```bash
cd backend
npm install
```
